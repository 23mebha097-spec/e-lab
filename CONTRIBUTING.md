# Contributing to E-Lab

Thank you for your interest in contributing to E-Lab! We welcome all developers to help make this 3D robotic environment better.

## How to Contribute

### 1. Set Up Your Environment
Follow the steps in the [README.md](README.md) to install dependencies:
```bash
pip install -r requirements.txt
```

### 2. Making Changes
- **Create a Branch**: Always create a new branch for your features or bug fixes.
  ```bash
  git checkout -b feature/your-feature-name
  ```
- **Code Style**: Please follow standard Python (PEP 8) guidelines.
- **Testing**: Run the existing `test_*.py` scripts to ensure your changes don't break the simulation.

### 3. Submitting a Pull Request
1. Commit your changes with a descriptive message.
2. Push your branch to the repository.
3. Open a Pull Request (PR) on GitHub.
4. Describe your changes clearly in the PR description.

## Reporting Issues
If you find a bug, please open an issue on the GitHub repository with:
- A description of the problem.
- Steps to reproduce it.
- Your OS and Python version.

## Contributor Recognition
All contributors who have their PRs merged will be listed in the [CONTRIBUTORS.md](CONTRIBUTORS.md) file!
